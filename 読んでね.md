# Expansion Studio

このフォルダには、Pokemon Emerald Expansion の個人用日本語化作業を補助する
GUI ツールを置いています。リポジトリ内のソースファイルを直接読み込みます。
ソースが定義元であり、ツールは解析できる範囲だけを保守的に編集します。
AIと共同作業で作成したアルファ版なので動作は保証しません。
尚、このプロジェクトは任天堂、ゲームフリーク、株式会社ポケモンとは無関係です。

## 起動に必要なもの

* Python 3.10 以降
* PySide6

リポジトリのルートで、初回だけ次を実行します。

```powershell
python -m pip install -r "workspace\requirements.txt"
```

起動コマンドです。

```powershell
python "ExpansionStudio.py"
```

ツールバーで `src` と `include` を含むリポジトリルートを選択してください。
まぁたぶんexe使ったほうが早いです。リリースにいるので。
https://github.com/koziyagoukei/ExpansionStudio/releases

## 現在できること

* **翻訳**: `src` / `include` の `_()` と `COMPOUND_STRING()` を検索し、
  文字列編集、検索、ファイル名フィルタ、未翻訳/変更済みフィルタ、文字数表示、
  CSV 入出力を行えます。イベントブラウザではリポジトリ配下の `.inc` を再帰検索し、
  `.string` 定義をラベル・本文・ファイル・行番号・使用数・定義種別で一覧表示します。
  `msgbox` / `message` のテキストをイベントツリーで確認でき、ラベル・本文の横断検索と
  相互参照に対応しています。INC の終端 `$` は編集時には隠し、保存時に必ず維持します。
  定義は「種別・相対パス・ラベル」の組み合わせで識別するため、INC と Poryscript の
  同名ラベルは推測で結び付けず未解決として表示します。対応済み `.string` と C 文字列は
  バックアップ付きで保存できます。
* **定数**: `include/constants` 配下の `#define` と enum 形式の定数を確認できます。
* **ファイル検索**: 識別子、文字列、アセットパスを横断検索できます。
* **ポケモン**: `[基本] [能力] [特性] [進化] [技] [図鑑] [画像]` のタブで確認・
  編集できます。タイプは日本語名で表示し、種族値合計、特性説明、進化先、
  レベルアップ技、正面画像、鳴き声 ID を確認できます。進化先や習得技を
  ダブルクリックすると対象データへ移動します。レベルアップ技、タマゴ技、
  教え技・マシン技は、実際の定義ファイルを選択して編集・差分確認・保存できます。
  進化定義も複雑な条件式を維持したまま編集できます。
* **わざ**: タイプ・分類・威力・命中・PP・効果を読みやすく表示します。接触、
  パンチ、音技、風技などはチェックボックスで編集できます。追加効果タブでは
  「10%でまひ」のような効果・確率・自分対象を編集でき、参照元も確認できます。
* **アセット**: 初期状態では PNG を持つ画像グループだけを表示します。同じベース名の
  `.gbapal` / `.pal` / `.bin` / `.4bpp` は一つの画像グループとして扱います。
  静的参照が 0 件のグループは「未使用候補」と表示され、パス順・未使用候補優先・
  参照数順で並べ替えできます。削除は行わず、グループ全体を
  `quarantine/assets/<日時>/` へ移動します。PNG のないグループはフィルターで表示します。
* **依存関係**: 定数、種族 ID、技 ID、ファイルパス、シンボルの静的な参照元を
  一覧できます。
* **Glyph Table**: 選択したフォント PNG を 8x16 セルの Glyph ID 単位で表示します。
  Glyph ID、表示文字、glyph プレビュー、幅、使用回数、使用箇所数を確認でき、
  Glyph ID / 文字検索と未使用 Glyph 抽出に対応します。通常文字の charmap 割り当ては
  バックアップ付きで追加・変更・削除できます。日本語・英語の自動プロファイル方式は
  使わず、選択中の Glyph ID をそのまま可視化・編集します。
* **Poryscript**: `.pory` を再帰検索し、ファイル名・本文・ラベルで検索、閲覧、
  編集、バックアップ付き保存できます。新規作成では空の Poryscript または
  `workspace/templates/poryscript` のテンプレートを選び、`{{SCRIPT_NAME}}` や
  `{{MESSAGE}}` などの置換変数をフォームに入力してプレビューできます。既存の
  `.inc` は元ファイルを変更せず、`raw` ブロックを含む `.pory` として保存できます。
  外部ツール未設定でも新規作成・編集・保存できます。設定済みなら stdout/stderr
  ログを確認しながらコンパイルできます。
* **設定**: poryscript.exe、source directory、output directory は Poryscript
  編集画面とは別の設定タブで管理します。

編集して保存する前には、対象ファイルと同じ場所に `.bak` バックアップを作ります。
すでに `.bak` がある場合は日時付きのバックアップを作ります。

画面右上の言語選択で、日本語と英語を切り替えられます。上部ツールバーには、
差分、保存、ビルド、ROM 起動、ビルド/ROM コマンド設定があります。先に
設定画面でコマンドを登録すると、選択したリポジトリルートを作業フォルダとして
実行します。

## 現在の制限

* C の完全な構文解析器ではありません。複雑なマクロや式は表示しても自動編集
  できない場合があります。
* アセットの参照確認は静的テキスト検索です。動的参照やビルド時生成物は保証
  できないため、削除後は必ず `make` で確認してください。
* 種族・技の追加は雛形の作成までです。内部 ID、enum、定数の再採番や削除は
  行いません。
* `.inc` 読込は安全性を優先し、最初は `raw` ブロックに包むだけです。既存の
  アセンブリを自動で高水準 Poryscript 構文へ置換しません。Poryscript の
  フローチャート表示と構造化 GUI 編集は未実装です。
* トレーナー、バトル AI、マップ専用エディタは未実装です。

## EXE 配布用ビルド

`em.png` はウィンドウと EXE のアイコンに使います。初回だけビルド用依存を入れ、
次を実行してください。

```powershell
python -m pip install -r "workspace\requirements-build.txt"
python "workspace\build_exe.py"
```

生成先は `workspace\dist\ExpansionStudio.exe` です。
EXE 版で初めて Poryscript テンプレートを開くと、同梱の初期テンプレートを
`workspace\dist\templates\poryscript` に展開します。EXE 版ではこのフォルダを
テンプレート管理の保存先として使用します。
